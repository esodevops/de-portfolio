Project overview

Data pipeline

From raw orders to dependable insight.

The workflow separates production ETL logic from exploration, then backs the result with validation scripts and automated tests so each stage remains repeatable.

01

Extract

Read the historical e-commerce CSV into Pandas with safe loading and clear logging.

02

Profile

Inspect columns, types, null values, duplicates, and inconsistencies in commerce records.

03

Clean

Standardize identifiers, dates, contact details, prices, quantities, and categorical values.

04

Normalize

Split the flat source into customers, products, addresses, orders, and payment methods.

05

Analyze

Apply joins, group calculations, ranking, running totals, cases, and window operations.

06

Load

Prepare structured outputs for PostgreSQL storage and reusable business reporting.

07

Validate

Run schema checks and pytest coverage to confirm required files and columns are present.

Database design

Five connected commerce tables.

The normalized model separates core entities while retaining the keys needed to connect every order to its customer, product, shipping destination, and payment outcome.

customersIdentity and contact details
productsCatalog, category, and price
shipping_addressCustomer delivery locations
ordersQuantity, value, and order date
payment_methodMethod and transaction status
