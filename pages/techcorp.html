Project overview

Resilient cloud, built to scale.

System design

Public entry. Private workloads. No loose ends.

Traffic enters through one controlled edge, while application and data tiers stay protected inside private subnets.

01

Application Load Balancer

Routes HTTP traffic and continuously checks the health of both web targets.

02

Web Server · AZ 1

Apache workload in a private subnet, reachable only through the load balancer.

03

Web Server · AZ 2

A second application node provides redundancy across availability zones.

04

PostgreSQL

Private database access restricted to the application security group on port 5432.

05

NAT Gateway

Allows private instances to fetch updates without accepting inbound internet traffic.

06

Bastion Host

Single administrative gateway protected by an allow-listed source IP.

Why it matters

Production-minded from the first apply.

Every layer is designed to solve a concrete production concern while maintaining security and reliability.

01

Fault-tolerant web tier

Application instances span two availability zones, with health-based traffic distribution at the edge.

02

Least-access networking

Only the ALB is public-facing. SSH and database access follow narrowly defined security-group paths.

03

Repeatable deployment

Terraform and startup scripts turn the complete environment into reviewable, versioned infrastructure code.
