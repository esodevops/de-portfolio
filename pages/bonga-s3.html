Bonga E-commerce Cloud Data Pipeline

Project overview

Move private ecommerce data without putting it in Git.

Bonga separates data delivery from application code. Teams can use safe demo files locally, then switch to protected cloud datasets through environment configuration.

The challenge

Secure cloud integration without compromising data trust.

Production datasets contain sensitive information that should never be committed to version control. The pipeline addresses this while maintaining reliable, repeatable data loading.

01

Secure ingestion

Production datasets and credentials stay outside the repository.

02

Provider flexibility

The same workflow supports Amazon S3, Azure Blob Storage, and Google Cloud Storage.

03

Reliable fallback

When cloud storage is not configured, synthetic local data keeps development reproducible.

04

Private data remains private

Credentials live in ignored environment files or CI secrets.

05

Providers are interchangeable

Teams can move between S3, Azure, GCS, and local demo data without changing the workflow.

06

Trusted outputs

Constraints, row counts, and orphan checks prove that the database is analysis-ready.

Delivery workflow

From protected files to a verified database.

The workflow is automated end to end and safe to rerun. Each stage prepares the data for the next, ending with database tables that are ready to query.

01

Configure

Environment profiles keep database and cloud settings outside the code.

02

Start

Docker Compose launches a consistent PostgreSQL service.

03

Fetch

S3, Azure Blob, or GCS securely delivers the selected CSV datasets.

04

Load

Tables are cleared and records reload in dependency order.

05

Validate

Row counts and orphan checks prove the load is complete and connected.

Database design

Four connected ecommerce tables.

Primary keys establish identity, foreign keys preserve relationships, and checks prevent negative prices or invalid quantities. This structure supports operational reporting without duplicating core entities.

productsProduct catalog with prices
customersCustomer identities and contacts
ordersOrder headers with dates
order_itemsLine items with quantities
