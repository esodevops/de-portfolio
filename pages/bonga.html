Project overview

Make ecommerce data dependable before analyzing it.

Bonga needs product, customer, order, and line-item data to agree. This project builds the controls around that data so every query starts from a consistent source of truth.

The challenge

Fragmented records weaken every decision downstream.

Manual entry and disconnected systems caused quality problems that made reports unreliable. The pipeline targets the issues with the greatest operational impact.

01

Fragmented source files

Four independent CSV files must be loaded in dependency-safe order.

02

Relationship risk

Orders must map to real customers, and every order item must map to both a valid order and product.

03

Repeatability

The same workflow must produce the same clean database locally and in continuous integration.

04

Bad relationships fail early

Primary, unique, foreign-key, and check constraints prevent silent corruption.

05

Idempotency

Every run begins cleanly with dependency-aware truncation.

06

Decision-ready analysis

The database must support sales, pricing, customer, category, and product-performance questions.

Data pipeline

From raw files to a verified database.

The workflow is automated end to end and safe to rerun. Each stage prepares the data for the next, ending with database tables that are ready to query.

01

Configure

Environment-specific settings keep credentials and ports outside the code.

02

Start

Docker Compose launches a consistent PostgreSQL service.

03

Model

SQL creates tables, constraints, foreign keys, and indexes.

04

Load

Tables are cleared and CSV records reload in dependency order.

05

Validate

Row counts and orphan checks prove the load is complete and connected.

Database design

Four connected ecommerce tables.

Primary keys establish identity, foreign keys preserve relationships, and checks prevent negative prices or invalid quantities. This structure supports operational reporting without duplicating core entities.

productsProduct catalog with prices
customersCustomer identities and contacts
ordersOrder headers with dates
order_itemsLine items with quantities
