Infrastructure and delivery

From source code to a running AWS storefront.

The platform makes each deployment stage explicit, repeatable, and verifiable.

01

Define

Terraform describes networking, EKS, IAM, RDS, DynamoDB, S3, and Lambda.

02

Build

Each component becomes a portable container image for x86-64 or ARM64.

03

Configure

Secrets and generated values connect workloads to AWS dependencies.

04

Deploy

Helm installs the services and exposes the UI through an AWS load balancer.

05

Verify

Health checks, end-to-end tests, metrics, and traces confirm the journey.

Application architecture

Small services, clear responsibilities.

The storefront remains one connected customer journey while each business capability owns its API and persistence choices.

Java / Spring Boot

Store UI

Renders the storefront, cart, checkout, topology, and optional AI assistant.

Go / MySQL

Catalog

Serves products and tags from a relational catalog database.

Java / DynamoDB

Cart

Maintains fast, session-aware shopping baskets.

Node.js

Checkout

Coordinates delivery, payment, and order submission.

Java / PostgreSQL

Orders

Persists completed purchases and emits order events.

EKS / ALB / S3

Platform

Runs containers, routes public traffic, and stores assets.

Operations and experimentation

More than a storefront.

Built-in operational features turn the application into a practical environment for platform demonstrations, troubleshooting, and reliability exercises.

01

Independent scaling

Kubernetes deployments, probes, resource settings, and autoscaling let busy services respond independently.

02

Built-in observability

Prometheus-compatible metrics and OpenTelemetry OTLP tracing help follow requests across service boundaries.

03

Resilience testing

Configurable chaos endpoints, retries, probes, and multiple backends create a useful reliability lab.

04

End-to-end testing

Cypress tests cover home, catalog, cart, and checkout, while a load generator drives sustained traffic.

05

Platform introspection

Metadata and topology pages expose runtime environment details, component health, and configured dependencies.

06

Generative AI integration

An optional chat experience supports Amazon Bedrock, OpenAI-compatible endpoints, or a mock provider.
