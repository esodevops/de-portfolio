Project overview

From raw files to reliable records.

MetroBank models the core activity of a retail bank across branches, customers, employees, accounts, transactions and loans. The focus is not simply storing data—it is loading it safely, protecting its quality and making every balance change traceable.

Data pipeline

A load order that respects relationships.

The import runs inside one transaction with fail-fast error handling. Temporary all-text staging tables isolate messy source data while transformations normalize values before production upserts.

01

Stage

Load six CSV files into disposable text-based staging tables.

02

Clean

Repair email, phone, account and transaction values.

03

Validate

Apply strict types, checks, uniqueness and required fields.

04

Upsert

Merge parent-first with conflict-safe, repeatable statements.

05

Commit

Publish all changes atomically—or roll the entire batch back.

Architecture

A connected banking model.

Foreign keys enforce the real-world links between each entity. Branches and customers are established first, accounts become the operational hub, and downstream financial activity stays anchored to its owner.

branchesBank locations and details
customersAccount holders and contacts

Production controls

Security and auditability included.

The database separates operational access from analytics access, builds permissions into future objects with default privileges and creates a permanent evidence trail for sensitive financial changes.

01

Role-based access

app_user receives CRUD privileges while readonly_analyst stays strictly SELECT-only.

02

Audit logging

An AFTER UPDATE trigger captures balance changes, the acting database user and the exact event time.

03

Indexed access paths

B-tree indexes support common customer-account and account-transaction lookup patterns.

04

Referential integrity

CASCADE updates and RESTRICT deletes preserve valid dependencies.

05

Domain constraints

UK phones, valid emails, positive amounts and controlled types.

06

Automatic timestamps

Update triggers maintain trustworthy modification metadata.
