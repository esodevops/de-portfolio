What it demonstrates

From sign-in to scale-out.

Every layer is designed to solve a concrete production concern while keeping the end-user experience simple.

01

Complete task workflow

Create, edit, complete, and delete personal todos through a clean, responsive interface.

02

Secure authentication

JWT sessions, bcrypt password hashing, protected routes, and httpOnly cookie support.

03

Redis caching

Frequently requested todo lists and usernames are cached with graceful failure fallback.

04

Elastic infrastructure

An Application Load Balancer and EC2 Auto Scaling Group keep the API available as demand changes.

05

Full observability

Structured application logs, CloudWatch dashboards, health checks, metrics, and SNS alarms.

06

Automated delivery

GitHub Actions tests, builds, and deploys both the frontend and containerized backend.

System design

One secure entry point. Purpose-built services behind it.

CloudFront serves the React application from a private S3 bucket and proxies API traffic to the load-balanced Go backend.

01

Frontend Delivery

React application served from S3 via CloudFront with API proxy routing.

02

Load Balancing

Application Load Balancer distributes traffic across EC2 instances with health checks.

03

Go API

Containerized Go backend with Gin framework running on Auto Scaling EC2 instances.

04

Data Layer

MongoDB Atlas for persistence with ElastiCache Redis for fast caching.

05

Infrastructure

Terraform manages VPC, subnets, security groups, and IAM policies.

06

Monitoring

CloudWatch logs, metrics, dashboards, and SNS alarms for 24/7 visibility.
